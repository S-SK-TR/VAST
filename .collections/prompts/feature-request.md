# ✨ Feature Request Prompt

> Yeni özellik talebi için sık kullanılan prompt şablonu.

---

## Kullanım

Bu prompt'u yeni özellik talep ederken kullanın:

```
## Yeni Özellik Talebi

### Özellik Adı
[Kısa, açıklayıcı bir isim]

### Problem / Motivasyon
[Bu özellik hangi problemi çözüyor? Neden gerekli?]

### Önerilen Çözüm
[Özelliğin nasıl çalışmasını bekliyorsunuz?]

### Kullanıcı Hikayesi
Bir [kullanıcı rolü] olarak,
[özellik/yetenek] istiyorum,
böylece [fayda/değer] elde edebilirim.

### Kabul Kriterleri
- [ ] [Kriter 1]
- [ ] [Kriter 2]
- [ ] [Kriter 3]

### Mockup / Tasarım (opsiyonel)
[Varsa görsel ekleyin]

### Alternatifler
[Düşünülen alternatif çözümler]

### Teknik Notlar
[Teknik kısıtlar, bağımlılıklar, dikkat edilecekler]

### Öncelik
- [ ] Must Have — Zorunlu
- [ ] Should Have — Önemli
- [ ] Could Have — İyi olur
- [ ] Won't Have — Bu iterasyonda değil
```

---

## Uygulama Süreci
1. Feature request oluştur
2. Backlog'a ekle (plans/backlog.md)
3. Implementation plan hazırla (plans/implementation-plan.md)
4. Gerekiyorsa ADR yaz (docs/decisions/)
5. Develop, test, document, deploy
